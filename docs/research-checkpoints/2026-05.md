### 2.2 MolmoSpaces multi-agent 状态

**结论：multi-agent 仍未到位**。

证据：
- Allen AI 官方 MolmoSpaces 介绍页（A 档，
  [allenai.org/blog/molmospaces](https://allenai.org/blog/molmospaces)）
  仍描述 single-arm / dual-arm 配置，**没有 multi-agent 机器人队列的支持
  申明**。
- arXiv:2602.11337（MolmoSpaces 论文，A 档）描述 MolmoSpaces-Bench 8 个
  任务，全部是单 agent 设定。
- Q14（2026-04.md §7）"MolmoSpaces 何时支持 multi-agent" **仍是开问**。

**对 H2 的影响**：H2 假设 "AI2-THOR（导航/领地）+ MolmoSpaces（清扫/抓取）"
是当前正确的双 substrate。本期没有证据反对 H2，但 **Phase 3 的多 agent
plan 仍被 substrate 卡住**——roboclaws 自己短期可走的路只有：

1. 在 AI2-THOR 上做多 agent（MAP-THOR / TBONE 路线）
2. 等 MolmoSpaces 上游 multi-agent（不知何时来）
3. 自研补丁（强求于 single-agent substrate 上叠多 agent 的协调层）

**触发器评估**：vectors 卡片把 "MolmoSpaces 多 agent 支持落地 *或* 公开表态
反对" 列为 off-schedule trigger。本期两者皆无，仅维持现状。

### 2.3 多 coding-agent 多 robot precedent（H8）

**结论：仍未出现严格匹配的 precedent，但邻接工作明显增多**。

具体匹配是 vectors 卡片 V6 "watchlist (the gap that makes H8 risky):
any paper, blog, or repo describing multiple coding-agents simultaneously
driving multiple simulated robots"——以下都是邻接但不完全匹配：

- **CooperBench**（arXiv:2601.13295，2026-01 起）——"Why Coding Agents
  Cannot be Your Teammates Yet"，首个度量多 coding-agent 协作的 benchmark。
  软件工程域，但**正是 H8 的同源问题（多 coding-agent 协调）的 SE 版本**。
- **InteractGen**（arXiv:2512.00797，2025-12）——LLM 驱动的多 agent 框架，
  部署到异构机器人队，3 个月开放使用研究。**这是多 agent 编排 + 多机器人
  的部署研究**，差只差"多 coding-agent"这一层语义——它的 agent 是
  perception/planning/decision/verification/reflection/delegation 这类
  分工，不是"几个 Cursor/Claude Code 同时跑"。
- **AAMAS 2026**（2026-05-25 至 29，Paphos 塞浦路斯）正在召开
  ——多篇多 agent 机器人编排相关。具体如 arXiv:2604.06629 "Logical Robots:
  Declarative Multi-Agent Programming in Logica"。AAMAS 论文流量较大，
  会后 6 月 checkpoint 应专门扫一遍。

**对 H8 的影响**：仍成立——严格的"多 coding-agent 同时驱动多机器人在
仿真中"目前仍是 frontier 位置，roboclaws 仍处于"无公开 precedent"的窗口
里。但是：

1. CooperBench 直接对应 H8 的 SE 半边——它的发现"agents cannot be your
   teammates yet" 对 roboclaws 是个重要预警，多 coding-agent 协作本身的
   失败模式可能在机器人场景下被进一步放大。
2. InteractGen 与 AAMAS 2026 的存在意味着"多 agent 多机器人"这件事已经
   被多个团队接近 frontier。**roboclaws 的 frontier 窗口可能在 2-3 个
   月内被关闭**，而不是更长。

**建议**：把 H8 从 "维持" 提升为 "重点观察"。下个 checkpoint 必须扫
AAMAS 2026 全部论文。

## 4. 8 个 Hidden Assumptions 的现状评估

每条结论的格式：**结论**（一行）+ **依据**（要点）+ **下次 checkpoint
应特别留意的反例信号**。

### H1：同 agent 既驱动机器人又做视觉 review

- **结论**：**强化**。维持假设。
- 依据：Opus 4.7 视觉跃迁（XBOW 54.5% → 98.5%，3.75MP，1M context）
  使得同 agent 视觉 review 的能力门槛被显著降低。Anthropic 自己描述
  Opus 4.7 "devises ways to verify its own outputs before reporting back"
  直接对应 H1 的"自评"语义。
- 反例信号：任何 robotics benchmark 上 separate-VLM-judge 显著优于
  same-agent visual review 的对照实验（vectors 卡片 off-schedule trigger）。

### H2：AI2-THOR + MolmoSpaces 是当前正确的双 substrate

- **结论**：**稳定**，维持假设。
- 依据：MolmoSpaces 仍是当前最有希望的 manipulation substrate；AI2-THOR
  仍是 navigation/territory 的可用 substrate。无替代信号。
- 反例信号：MolmoSpaces 公开放弃 multi-agent / 出现一个公开支持
  multi-agent 的 alternative substrate（Genesis 1.0、Habitat 4、
  ManiSkill 3 多 agent 等）。

### H3：MCP + SKILL.md 是 Mode 1 + Mode 3 的正确打包

- **结论**：**稳定，且支撑面增强**。维持假设。
- 依据：MCP 2026 Roadmap 正常推进；Skills open standard 落地 32 个工具；
  三层协议（MCP + A2A + WebMCP）成行业共识。
- 反例信号：per-framework custom interface 在 robotics benchmark 上显著
  优于 MCP + SKILL.md。

### H4：四模式共存，Mode 3 为中心

- **结论**：**正向**。维持假设。
- 依据：OpenClaw governance 推进、Anthropic Managed Agents 成熟（含
  memory + multi-agent orchestration）、Codex Routines GA——四模式的
  上游基础设施都在按预期成熟。
- 反例信号：OpenClaw foundation 治理停滞 / Mode 3 主要 driver SDK
  （Claude Code、Codex）出现质变阻塞 / 一个新 driver 在质上显著超过
  现有 driver 使得 mode 4 形态 collapse。

### H5：公私评估真相分离是正确的能力暴露方式

- **结论**：**未受挑战**。维持假设。
- 依据：本期没有 peer 项目暴露同款 semantic profile 模式，也没有
  counter-example。PR #112 是 H5 第一次产品化的真实机器人路径——还在
  draft 阶段，尚未产生证伪机会。
- 反例信号：任何项目把 ground truth 暴露给公共 agent metadata 而仍能
  产生干净评估的实例。

### H6：静态 HTML 报告是正确的证据面

- **结论**：**未受挑战**。维持假设。
- 依据：本期没有 substrate 内建 proof-bundle 出现。Allen AI、NVIDIA Isaac
  Sim、HuggingFace LeRobot 都没有做相当于 roboclaws HTML report 的对应
  artifact。
- 反例信号：任何 substrate 推出"approval pack" / "proof bundle" 的原生
  抽象。

### H7：多 provider 混用是正确的实验设计

- **结论**：**强化**。维持假设。
- 依据：Opus 4.7 + Kimi K2.6 + GPT-5.4 / 5.5 / Codex 都在 4 周内取得
  meaningful 进步——multi-provider 选项的真实质量上限和差异化都显著
  提升。Mock + MiMo + NVIDIA + Anthropic + OpenAI + Kimi 多线挂载是有
  科学价值的实验设计。
- 反例信号：某个 provider 显著吞掉其它（在 robotics 相关任务上）。

### H8：多 coding-agent 多机器人是无 precedent 的 frontier

- **结论**：**仍成立但缓冲期不长**。建议升级为重点观察。
- 依据：本期 CooperBench、InteractGen、AAMAS 2026 多个邻接工作出现；
  严格匹配的 precedent 仍未出现。但 frontier 的"墙"在被多个团队同时
  从不同方向逼近。
- 反例信号：**任何**一篇明确描述"multiple coding-agents simultaneously
  driving multiple simulated robots" 的论文 / 博客 / 仓库出现（这条
  vectors 卡片已经显式列为 off-schedule trigger）。
