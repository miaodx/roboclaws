<domain>
## Phase Boundary

Answer two product questions with data rather than intuition:

1. Does a structured occupancy-grid-style overhead beat the current photo overhead?
2. Does a chase-cam materially improve navigation decisions beyond map-v2 alone?

**In scope:**
- Three prompt-image variants:
  - `baseline` = FPV + current photo overhead
  - `map-v2` = FPV + structured overhead
  - `map-v2+chase` = FPV + structured overhead + chase-cam
- Shared view primitives: structured map renderer, chase-cam helpers, view-builder dispatch.
- `--views` rollout across `examples/openclaw_demo.py`, `examples/territory_game.py`, and `examples/coverage_game.py`.
- `examples/view_experiment.py` harness + `scripts/analyze_view_experiment.py`.
- Local-dev Kimi sweep + NVIDIA confirm + decision record.

**Out of scope:**
- Phase 2.6 autonomous MCP flow (`examples/openclaw_nav_autonomous.py`). Phase 2.4 stays on the push-model demo/game paths.
- Shipping the winning variant as the default (`A-05` remains a follow-up phase).
- GPT-4o / Claude Sonnet full-grid sweeps.
- Any action-space change or game-rule change.

</domain>

<decisions>
## Locked Planning Decisions

### D-01: Single-agent OpenClaw first
- `02.4-01` lands the shared view surface on `examples/openclaw_demo.py` before territory/coverage.
- Reason: this is the narrowest end-to-end seam for validating the new prompt-image contract without dragging multi-agent game logic into the first implementation step.

### D-02: Stay on the Phase-2.4 push-model seam
- The "one agent first" work means `examples/openclaw_demo.py`, not the autonomous Phase-2.6 MCP loop.
- Do not couple this phase to `examples/openclaw_nav_autonomous.py` or Phase 2.6's tool-surface decisions.

### D-03: A-01 and A-02 are already resolved
- Image payload flow-through and coverage semantics were shipped before ingest (`ddfb523`, 2026-04-15) and confirmed stale-as-warning on 2026-04-20.
- Do not spend a Phase 2.4 plan re-litigating them.

### D-04: Four-plan decomposition
- `02.4-01` shared view primitives + `openclaw_demo.py`
- `02.4-02` territory + coverage rollout
- `02.4-03` `NvidiaProvider` + harness
- `02.4-04` analysis + local-dev study + writeup/retro

</decisions>
