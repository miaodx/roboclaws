# Phase 02.4 Plan 01: OpenClaw Demo View Variants Summary

**Landed the shared Phase-2.4 view primitives on the smallest real seam first: the push-model `examples/openclaw_demo.py` path now supports `baseline`, `map-v2`, and `map-v2+chase` through a single shared builder.**

## Accomplishments

- Added `GameVisualizer.render_structured_map(...)` so the repo can render reachable/unreachable cells, claimed or covered overlays, and heading-aware agent markers without relying on a photoreal overhead frame.
- Added chase-camera helpers to `MultiAgentEngine` for registering, updating, and reading a third-person frame tied to an agent pose.
- Created `roboclaws/core/views.py` with shared variant validation plus prompt-image assembly for all three view modes.
- Wired `examples/openclaw_demo.py` to accept `--views {baseline,map-v2,map-v2+chase}` and route prompt images through the shared builder.

## Verification

- `pytest -q tests/test_visualizer_structured.py tests/test_engine_chase_cam.py tests/test_views.py` - passed
- `python examples/openclaw_demo.py --help | rg -- --views` - passed

## Notes

- This work intentionally stayed on ndarray prompt images only. Base64 encoding for direct-VLM call sites was left for the later game rollout and harness work.
- Live chase-camera validation against a real AI2-THOR render remains part of the local-dev gate in `02.4-04`; this plan only claims mocked math and cloud-safe wiring.

No per-task commits were created in this cloud session. The implementation is present in the current working tree and covered by the targeted checks above.
