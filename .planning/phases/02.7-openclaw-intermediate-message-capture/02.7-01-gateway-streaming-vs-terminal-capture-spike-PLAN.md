De-risk transcript capture before touching production runtime code. Probe the pinned Gateway's real `POST /v1/chat/completions` surface in both modes:

1. `stream=true` — do we receive usable assistant-visible text events before the request completes?
2. `stream=false` — what can terminal-body capture recover after the response finishes?

This plan does not implement transcript capture in the shipped bridge. It produces the evidence and decision record that Plan 02 must obey.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02.7-02 | Tampering | Verdict based on synthetic or stale evidence | mitigate | Require dated live probe runs against the pinned Gateway surface and record exact output file paths plus model/image context in `02.7-SPIKE-FINDINGS.md`. |
