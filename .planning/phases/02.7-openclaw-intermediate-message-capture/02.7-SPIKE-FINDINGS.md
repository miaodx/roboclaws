## Decision

`streaming-primary`

The pinned Gateway surface emits parseable assistant text deltas before completion, and it did so with far more than the minimum two non-empty fragments required by the plan. Terminal-body capture remains worth preserving as an explicit fallback mode, but it is not the primary runtime contract.

## Why the loser lost

`terminal-body-primary` lost because it cannot answer the core operator question: what the assistant said between tool calls. It only recovers a single terminal body after completion, with no intermediate boundaries and no timing. That makes it truthful but insufficient as the default path for Phase 2.7.

## What Plan 02 must implement

1. `roboclaws/openclaw/bridge.py` should default to `stream` mode at module scope, with a comment pointing back to this spike.
2. The bridge should collect streaming transcript entries from `choices[0].delta.content` and timestamp them relative to run start.
3. The bridge must still support an explicit `terminal-body` mode and persist that source truthfully as a fallback contract.
4. `run_result.json` must record both the requested mode (`transcript_capture_mode`) and the actual source used (`transcript_source`).
5. `trace.jsonl` should append additive `assistant_transcript` events so the renderer can line transcript entries up with tool traffic.
6. Do not auto-retry a live run in a second mode. Mode is chosen once before kickoff.
