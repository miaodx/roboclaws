## Objective

Implement the chosen transcript capture path additively in the autonomous runtime. Persist intermediate assistant messages with timestamps and source metadata so downstream renderers can show them without inferring from the final response body.

## Must Be True

- The chosen primary route from plan `02.7-01` is implemented.
- Captured messages persist additively into `trace.jsonl`, `run_result.json`, or a clearly-linked companion file.
- Each message records enough metadata to correlate it with the run:
  - timestamp
  - source (`stream` or `terminal-body`)
  - content
  - any chunk/order index needed for reconstruction
- Old autonomous traces remain renderable.
