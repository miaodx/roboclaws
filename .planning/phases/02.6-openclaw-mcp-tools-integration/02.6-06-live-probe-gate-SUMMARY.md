## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] MCP server `host="127.0.0.1"` unreachable from Gateway container on Linux**
- **Found during:** Task 2 (Probe 1 — MCP initialize reachability)
- **Issue:** Plan 01's T-02.6-01 mitigation text claims "Docker's `host-gateway` routing to `host.docker.internal` still works against loopback on Linux." Live testing on Linux kernel 6.17 + Docker 29.2.1 proved this false — `host.docker.internal` resolves to docker0 bridge IP (172.17.0.1), which cannot reach host `127.0.0.1`. First Probe 1 attempt logged `[bundle-mcp] failed to start server "roboclaws": TypeError: fetch failed` and the container could not reach the MCP server.
- **Fix:** Changed `examples/openclaw_nav_autonomous.py` explicit `host=` from `"127.0.0.1"` to `"0.0.0.0"`. Matches the spike (`spike/mcp_image_probe.py:30`), the Phase 2.5 retrospective, and `docs/openclaw-local.md:244`. Verified with a trivial `HTTPServer('0.0.0.0', 18788)` probe that the container reached it via `host.docker.internal` as expected.
- **Files modified:** `examples/openclaw_nav_autonomous.py` (lines 214-223)
- **Verification:** Probe 1 re-run showed HTTP_STATUS=200 + body with `serverInfo.name=roboclaws`, zero MCP-bundle errors post-fix.
- **Committed in:** `a3e6332`
