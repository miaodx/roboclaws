The `minimal` baseline is consistent with the spike (+2.5%). The `coding` baseline has
dropped 26% (15,396 → 11,335) on the same Gateway image `ghcr.io/openclaw/openclaw:2026.4.14`.
Two possible explanations:
1. The 3-tool surface (`observe`+`move`+`done`) inflates `minimal` relative to the single-tool
   spike (2.5% growth consistent with that).
2. The `coding` profile's tool catalog may have been slimmed between the spike's measurement
   and today's — or the spike's 15,396 figure was measured under a slightly different
   Gateway configuration (e.g. different `timeoutSeconds`, different agent-model pairing)
   that is no longer reproducible.

Result is **stable and reproducible**: the identical result under both `anthropic_kimi`
custom provider and `kimi/k2p5` plugin provider shows the number is not a provider-specific
artifact.
