`docs/retrospectives/phase-2.5-local-dev.md` for future reference.

<action>
The goal is to run TWO separate autonomous runs in sequence WITHOUT
killing the Gateway container between them. Plan 02.5-05's example
bootstraps AND tears down the Gateway each run — so for this probe we
MUST bypass the example's own Gateway bootstrap/teardown.

Option A — bring up the Gateway once, then run the inner logic twice
via a short script:

1. Bootstrap the Gateway once:
   ```bash
   TOKEN=$(./scripts/openclaw-bootstrap.sh)
   echo "$TOKEN" > /tmp/openclaw-token
   ```
2. Run the example with a modified environment that skips its internal
   bootstrap + container-remove (simplest: wrap the example's core
   logic in a reusable Python helper, or use `OPENCLAW_GATEWAY_URL`
   env to point at the existing Gateway and MANUALLY skip the
   bootstrap+teardown within the example).
   If the example doesn't expose a `--skip-bootstrap` flag yet, this
   probe requires a 5-line patch to the example:
   ```python
   parser.add_argument("--skip-bootstrap", action="store_true",
                       help="Use an existing Gateway at OPENCLAW_GATEWAY_URL "
                            "(for back-to-back runs against a long-lived Gateway)")
   ```
   When `--skip-bootstrap` is True: read TOKEN from
   `OPENCLAW_GATEWAY_TOKEN` env; do NOT invoke openclaw-bootstrap.sh;
   do NOT `docker rm -f openclaw-gateway` in the finally block.
   This flag is small and reasonable addition; file a follow-up patch
   inside this task's action if the example doesn't have it.
3. Run A:
   ```bash
   OPENCLAW_GATEWAY_TOKEN="$(cat /tmp/openclaw-token)" \
     python examples/openclaw_nav_autonomous.py \
     --skip-bootstrap --scene FloorPlan201 \
     --max-moves 20 --wall-budget 120
   ```
4. Capture run A's run_dir as $RUN_A. Extract the agent's final
   reasoning from summary.json:
   ```bash
   jq -r '.final_message' $RUN_A/summary.json > /tmp/run-a-final.txt
   ```
5. Immediately run B (no teardown between):
   ```bash
   OPENCLAW_GATEWAY_TOKEN="$(cat /tmp/openclaw-token)" \
     python examples/openclaw_nav_autonomous.py \
     --skip-bootstrap --scene FloorPlan201 \
     --max-moves 20 --wall-budget 120
   ```
6. Capture run B's run_dir as $RUN_B. Extract run B's first observe
   response and its agent reasoning:

   ```bash
   python -c "
   import json
   with open('$RUN_B/trace.jsonl') as f:
       events = [json.loads(l) for l in f if l.strip()]
   first_obs_resp = next(
       e for e in events
       if e.get('tool') == 'observe' and e.get('event') == 'response'
   ```

      continuation of run A ("continuing from where I left off, the
      kitchen is behind me").

   c. The docker exec rm -rf fired for agent-0's state/ on run B's
      start_run — verify the state/ dir is empty just before run B

