Base URL for all three tools: `http://host.docker.internal:18788`.

Content for each tool (format-agnostic — wrap according to T49's finding):

**observe**
- HTTP: `GET /observe`
- Request body: (none)
- Response body:
  ```json
  {"fpv": "<base64 JPEG 320x240>",
   "overhead": "<base64 JPEG 320x240>",
   "state": {"agent_id": <int>, "position": {...}, "rotation": {...},
             "camera_horizon": <float>, "last_action_success": <bool>},
   "human_message": "<string or null>"}
  ```
- Usage: *Call `observe` when you need to see the world — returns the
  agent's first-person view, an overhead map, and current physical
  state. Start every run with one `observe`.*

**move**
- HTTP: `POST /move`
- Request body:
  ```json
  {"direction": "<one of: MoveAhead | MoveBack | MoveLeft | MoveRight |
                        RotateLeft | RotateRight | LookUp | LookDown>"}
  ```
- Response body:
  ```json
  {"state": {...same shape as /observe.state...},
   "human_message": "<string or null>",
   "server_warning": "<string, only if present>"}
  ```
  Note: response deliberately does NOT include images — call `observe`
  again to see the new view.
- Usage: *Call `move` to take one physical step in the world. The
  response gives you the new state but NOT a new image — chain with
  `observe` if you need to re-look.*

**done**
- HTTP: `POST /done`
- Request body:
  ```json
  {"reason": "<short natural-language reason>"}
  ```
- Response body:
  ```json
  {"status": "ok", "reason": "<same reason>"}
  ```
- Usage: *Call `done` when the navigation goal is achieved or you are
  stuck and no further moves will help. This ends the run cleanly.*

After the three tool declarations, write the usage summary as a final
paragraph at the end of the `## Tools` section (verbatim):

> "Call `observe` when you need to see the world. Call `move` to take
> a physical step. Call `done` when the navigation goal is achieved or
> you're stuck."

Preserve all existing SKILL.md content. Do NOT remove the existing
"What this skill does" or "Play styles" sections — this is additive.

```markdown
## Human messages

Tool responses (from `observe` and `move`) may include a `human_message`
field. When present, it is a directive from a human observer watching the
run. Treat it as a high-priority hint — follow it when compatible with
your current goal, and acknowledge it explicitly in your reasoning
either way. If you cannot follow it, say why; if you choose to follow it,
say so. The human is steering you, not interrupting you.

Example (inside a tool response):

```json
{"state": {...}, "human_message": "try rotating left to see the window"}
```

Your next reasoning step should reference the message — e.g.
*"human observer suggests rotating left; compatible with exploring the
east wall, rotating now"* — before calling the next tool.
```

This section is the DURABLE record. The kickoff prompt from Plan 02.5-05
will point here rather than duplicate the contract.
</action>

This step runs on the local workstation only (requires real KIMI_API_KEY
and Docker). It is the durable validation that the declaration format
from the T49 spike actually results in the agent discovering and using

1. Start the sim_server on port 18788 standalone (no AI2-THOR required
   for this probe — use a stub that returns static 1x1 black JPEGs for
   /observe and logs every incoming request):
   ```bash
   python -c "
   from roboclaws.openclaw.sim_server import SimHTTPServer
   from unittest.mock import MagicMock
   import numpy as np, pathlib, time
   eng = MagicMock()
   eng.get_agent_state.return_value.frame = np.zeros((1,1,3), dtype=np.uint8)
   eng.get_agent_state.return_value.position = {'x': 0, 'y': 0, 'z': 0}
   eng.get_agent_state.return_value.rotation = {'x': 0, 'y': 0, 'z': 0}
   eng.get_agent_state.return_value.camera_horizon = 0.0
   eng.get_agent_state.return_value.last_action_success = True
   eng.get_overhead_frame.return_value = np.zeros((1,1,3), dtype=np.uint8)
   run_dir = pathlib.Path('output/openclaw-autonomous/skill-probe')
   run_dir.mkdir(parents=True, exist_ok=True)
   srv = SimHTTPServer(eng, agent_id=0, run_dir=run_dir, port=18788)
   time.sleep(600)
   " &
   SIM_PID=$!
   ```
2. Bootstrap the Gateway:
   `TOKEN=$(./scripts/openclaw-bootstrap.sh)`
   (Requires Plan 02.5-06 to have landed for the --add-host flag. If
   running this task before Plan 02.5-06 ships, use `--network host` on
   Linux manually.)
3. Fire a kickoff prompt via curl (single blocking call):
   ```bash
   curl -sS --max-time 60 -X POST "http://127.0.0.1:18789/v1/chat/completions" \
       -H "Authorization: Bearer $TOKEN" \
       -H "Content-Type: application/json" \
       -d '{"model":"openclaw/agent-0",
            "messages":[{"role":"user","content":
            "Use the observe tool declared in your skill. Do NOT describe the tool — actually call it."}]}'
   ```
4. Assert that the sim_server's trace.jsonl at
   `output/openclaw-autonomous/skill-probe/trace.jsonl` gained a row
   with `tool=observe, event=request` within 30 seconds of the kickoff:
   ```bash
   deadline=$((SECONDS + 30))
   while (( SECONDS < deadline )); do
     if grep -qE '"tool":\s*"observe".*"event":\s*"request"' \
         output/openclaw-autonomous/skill-probe/trace.jsonl 2>/dev/null; then
       echo "OK: agent called observe within 30s"
       break
     fi
     sleep 1
   done
   ```
5. On failure (no observe call in 30 s): dump the Gateway's response body
   and the container logs; the SKILL.md declaration format chosen in
   task 01 is NOT parsed by the Gateway. Revisit the T49 spike and
   adjust the ## Tools block to the actual format.
6. Cleanup: kill `$SIM_PID`, remove the container with
   `docker rm -f openclaw-gateway`.
</action>
