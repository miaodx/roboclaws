<domain>
## Phase Boundary

Invert the OpenClaw integration. Current push model (Python shoves FPV+overhead to the agent every step via `/v1/chat/completions`, one-shot JSON action) wastes OpenClaw's full agentic shell. Phase 2.5 tests the inverse: one kickoff prompt with a long wall-clock budget, and the agent pulls `observe` / `move(direction)` / `done(reason)` tools from a local HTTP server as needed. Human interjection piggybacks on tool responses via a stdin queue.

Scope: **single-agent, single-scene, one new example script** (`examples/openclaw_nav_autonomous.py`). Does NOT touch `bridge.step()` or any push-model caller. Parallel architectural bet to Phase 2.4 — if this shape wins, Phase 2.4's A/B is re-run inside it in a follow-up phase; in v1 they don't share code paths.

Out of scope (deferred): multi-agent autonomous, territory/coverage games under this shape, drawing/visualization response tools, websocket UI, SOUL-specific tuning.

</domain>

<decisions>
## Implementation Decisions (all LOCKED in source plan)

### Transport / routing
- **Tool server**: in-process stdlib `http.server.ThreadingHTTPServer` at `localhost:18788` (no new dependency).
- **Container → host**: `--add-host=host.docker.internal:host-gateway` default (Docker 20.10+, cross-platform). `--network host` is the Linux-only fallback, documented only in Error & Rescue.
- **Kickoff transport**: one `POST /v1/chat/completions` with `model=openclaw/agent-<id>`, per-call `timeout=wall_budget_s + 60`. MUST NOT mutate the shared bridge client's 180 s default (regression-tested).

### Tool contract (three tools)
- `GET /observe` → `{fpv, overhead, state, human_message}`, JPEG q=70 @ 320×240, matches push-path encoding. Writes a `frame_capture` trace event with `seen_by_agent: true` and resets move-since-observe state.
- `POST /move` with body `{direction, reason?}` validated against `NAVIGATION_ACTIONS`. Returns new state + pending `human_message`. Server captures FPV+overhead server-side for the replay (`seen_by_agent: false`) but does NOT send images back to the agent.
- `POST /done` with body `{reason}` — idempotent, sets `done_event`.
- `/move` before first `/observe` → 200 + `server_warning: "move before first observe"`. Non-fatal.

### Workspace isolation
- Per-run state reset: `start_run` wipes `/home/node/.openclaw/workspaces/agent-<id>/state/*` before the kickoff POST. Matches logged `[fixed-session-prefix-leaks-memory]` pitfall. Regression-tested.

### Human-message queue
- `collections.deque(maxlen=10)`. Oldest silently drop when full, BUT a `queue_overflow` trace event is written so drops are auditable.
- Stdin reader thread detects `sys.stdin.isatty() == False` and skips startup cleanly.

### Trace / replay
- Single `trace.jsonl` under `output/openclaw-autonomous/<run-id>/`. Every tool call + every frame capture logged with ts/event/wallclock.
- Frame-capture events distinguish `seen_by_agent: true` (from `/observe`) from `seen_by_agent: false` (server-side during `/move`) and classify move steps as `fresh_observe`, `reasoned_batch`, or `blind_batch`.
- `report.html` is a decision audit: every step shows FPV + overhead together, keeps 👁 + green border = seen and 🚶 + grey dashed border = unseen, and labels repeated moves as either reasoned continuation or unjustified blind batch.

### Lifecycle
- All internal threads `daemon=True`. Example wraps full run in `try/finally` so SIGINT tears down sim server → Gateway container → stdin thread in order.
- Wall-clock cap is the hard ceiling — if the blocking `/v1/chat/completions` exceeds budget, client-side kill, partial results reported.

### Environment
- **This phase is local-dev throughout.** Per updated `CLAUDE.md § Cloud vs local development`, a local session owns the full loop (bounded changes + real-hardware probes). T49 spike, T51 skill validation, and T55 (6 probes) all run locally with real Docker + real Kimi. No cloud handoff planned.

</decisions>
