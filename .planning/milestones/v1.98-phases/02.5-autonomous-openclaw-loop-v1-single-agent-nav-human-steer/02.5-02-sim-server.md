# Plan 02.5-02: SimHTTPServer — local tool server with frame-capture trace

**Source task in reviewed plan:** T50 (see `docs/retrospectives/plans/phase-2.5-autonomous-nav.md`)
**Owner:** cloud
**Wall estimate:** 3-4 hrs

## Objective

Ship the in-process HTTP tool server at `localhost:18788` that the
Gateway-side agent calls via `observe`, `move`, `done` tools. Wraps
stdlib `http.server.ThreadingHTTPServer` (no new dependency). Serializes
every AI2-THOR call through a single controller mutex. Emits a
`trace.jsonl` stream that distinguishes frames the agent saw
(`seen_by_agent=true`) from server-side snapshots taken during `/move`
(`seen_by_agent=false`) and classifies move steps as fresh-observe,
reasoned-batch, or blind-batch, so the replay in Plan 02.5-07 can
visualize decision cadence at a glance.

<action>
Create `roboclaws/openclaw/sim_server.py` with `from __future__ import annotations`
at the top. Module implements:

1. **Class signature**:
   ```python
   class SimHTTPServer:
       def __init__(
           self,
           engine: MultiAgentEngine,
           agent_id: int,
           run_dir: pathlib.Path,
           host: str = "127.0.0.1",
           port: int = 18788,
       ) -> None: ...
   ```
   Where `run_dir` is `output/openclaw-autonomous/<run-id>/` and the
   trace file is written to `run_dir / "trace.jsonl"` (created if absent).

2. **Routes** (stdlib `http.server.BaseHTTPRequestHandler`
   subclass wrapped in `ThreadingHTTPServer`):

   - `GET /observe` → 200 JSON with keys:
     ```json
     {"fpv": "<base64 JPEG 320x240 q=70>",
      "overhead": "<base64 JPEG 320x240 q=70>",
      "state": {"agent_id": <int>, "position": {...}, "rotation": {...},
                "camera_horizon": <float>, "last_action_success": <bool>},
      "human_message": <str or null>}
     ```
     FPV is `engine.get_agent_state(agent_id).frame`; overhead is
     `engine.get_overhead_frame()`. Both encoded via the same path as
     `roboclaws.openclaw.bridge._ndarray_to_data_url` but returning ONLY
     the base64 string (not the `data:image/jpeg;base64,` prefix).
     Also writes a `frame_capture` trace event with `seen_by_agent=true`.

   - `POST /move` body: `{"direction": "<action>", "reason": "<optional brief rationale>"}`. Validation:
     if `direction not in NAVIGATION_ACTIONS` → 400 JSON
     `{"error": "invalid direction", "valid": NAVIGATION_ACTIONS}`.
     Valid: call `engine.step(agent_id, direction)` UNDER the controller
     mutex. Response JSON:
     ```json
     {"state": {...same shape as /observe state...},
      "human_message": <str or null>,
      "server_warning": <str or absent>}
     ```
     The response body MUST NOT include `fpv` or `overhead` keys —
     images are captured server-side only, not sent back.
     After the step lands, capture FPV + overhead server-side and write
     a `frame_capture` trace event with `seen_by_agent=false` carrying
     the images, the optional `move_reason`, and a `decision_mode`
     field. `decision_mode` is:
     - `fresh_observe` for the first move after the latest `/observe`
     - `reasoned_batch` for repeated moves that carry a non-empty `reason`
     - `blind_batch` for repeated moves without a reason
     This is the key mechanism that gives Plan 02.5-07's replay smooth
     coverage between observations without losing the reasoning shape.

   - `POST /move` before any `/observe` (tracked via an instance bool
     `self._observed_once`): response still 200 with a
     `server_warning: "move before first observe"` field AND a
     `server_warning` trace event. Non-blocking: the skill's kickoff
     prompt already instructs "start by calling observe".

   - `POST /done` body: `{"reason": "<str>"}`. Sets
     `self.done_event: threading.Event` (attribute accessible from
     outside). Idempotent: second call returns 200, event stays set, no
     error. Response body: `{"status": "ok", "reason": "<same str>"}`.

3. **Thread-safety + shared state**:
   - `self._controller_lock = threading.Lock()` wraps every call into
     `engine.step()` and `engine.get_agent_state()` and
     `engine.get_overhead_frame()`. AI2-THOR is not reentrant.
   - `self._human_queue: collections.deque[str] = deque(maxlen=10)`.
     Stdin thread (created by Plan 02.5-05) appends via
     `server.enqueue_human_message(msg)`. Tool handlers call
     `server._pop_human_message()` which is `self._human_queue.popleft()
     if self._human_queue else None`. Deque's maxlen behavior silently
     drops the oldest entry when full; on every drop, write a
     `queue_overflow` trace event with the dropped message's first 80
     chars. Detect overflow by comparing `len(queue) == maxlen` BEFORE
     append.

4. **Lifecycle**:
   - ThreadingHTTPServer started on a background thread marked
     `daemon=True`.
   - `self.close()` calls `self._httpd.shutdown()` then
     `self._server_thread.join(timeout=2.0)`. Log a WARNING if
     `self._server_thread.is_alive()` after join (soft hard-stop; stdlib
     doesn't give us a real kill).
   - Support context-manager protocol: `__enter__` returns self,
     `__exit__` calls close().

5. **Trace writer**: one JSONL row per event. Schema:
   ```json
   {"ts": <float unix>, "tool": "observe|move|done|<none>",
    "event": "request|response|frame_capture|queue_overflow|server_warning",
    "request": {...or omit}, "response": {...or omit},
    "wallclock_elapsed": <float seconds since server start>,
    "seen_by_agent": <bool, only on frame_capture>,
    "decision_mode": "fresh_observe|reasoned_batch|blind_batch",
    "move_reason": "<str or omit>",
    "fpv": <b64, only on frame_capture>,
    "overhead": <b64, only on frame_capture>,
    "agent_state": {...only on frame_capture}}
   ```
   Writes are line-buffered (`open(path, "a", buffering=1)`).
   Serialize to `trace.jsonl` under a second lock
   (`self._trace_lock`) so concurrent handlers don't interleave lines.

6. **Imports**: `from __future__ import annotations` at the top.
   Type-hint public APIs. No new third-party deps — stdlib `http.server`,
   `threading`, `collections`, `json`, `base64`, `io`, `pathlib`, `time`.
   PIL is already an existing dep; reuse it for JPEG encoding via the
   same pattern as `bridge._ndarray_to_data_url`.

7. **Line length 100, ruff clean, type annotations on public APIs**,
   per CLAUDE.md.
</action>

## Rescue Paths

| Failure | Rescue |
|---|---|
| Port 18788 collides with another process during integration | SimHTTPServer accepts `port` kwarg; tests use a random high port or `port=0`. In-process runs on 18788 by convention; if already bound, fail early with a clear `OSError [Errno 98]` surfaced as a RuntimeError. |
| Agent calls `move` with invalid direction | /move returns 400. Skill agent sees the error in its next tool response and should retry with a valid direction. Logged as a warning event in trace.jsonl. |
| Agent calls `move` before any `/observe` | 200 + `server_warning` field in response AND a `server_warning` trace event. Non-fatal; the agent can still observe later. |
| Human message queue overflows | Hard cap at 10 queued messages; oldest silently drop (`deque(maxlen=10)`); `queue_overflow` trace event records the drop. Human interjection is best-effort + auditable. |
| Concurrent /observe + /move race AI2-THOR | `self._controller_lock: threading.Lock` serializes every engine call. Tests assert via instrumented mock. |
