## Accomplishments

- `examples/openclaw_nav_autonomous.py` now imports `make_roboclaws_mcp` + `RoboclawsMCPServer` from `roboclaws.openclaw.mcp_server`. The example is the first caller of plan 01's public factory outside its own test file; wave 1 plumbing is load-bearing.
- `run_autonomous_navigation` instantiates the MCP server via the factory, calls `.run_in_thread()` before the bootstrap subprocess, and tears it down via `.close()` in the `finally` block in the correct order (watchdog -> mcp_server -> gateway container -> stdin -> bridge -> engine).
- The watchdog and stdin threads both address `mcp_server` directly using the same method names they used on `SimHTTPServer` (`snapshot_metrics`, `write_runtime_event`, `enqueue_human_message`). Plan 01 intentionally preserved these names — this plan proves they're drop-in swaps.
- Kickoff prompt shrunk from 38 lines (13 non-empty) to 7 lines (5 non-empty). It names the three prefixed MCP tools (`roboclaws__observe`, `roboclaws__move`, `roboclaws__done`), delegates loop mechanics to `skills/ai2thor-navigator/SKILL.md`, interpolates `max_moves`, and preserves the `human_message` acknowledgment contract from D-10.
- Every Phase-2.5 escape-hatch substring is gone from the prompt: `curl`, `exec`, `/tmp`, `image tool`, `data:image`, `base64`, `host.docker.internal`, `18788`, `HTTP`. A single kickoff-prompt test asserts both the line budget AND the forbidden-substring list so a future edit that widens either fails loudly.
- `env["SIM_SERVER_URL"] = ...` replaced with `env.setdefault("ROBOCLAWS_MCP_URL", "http://host.docker.internal:18788/mcp")`. Operator-supplied values (e.g. for plan 06's local-probe gate pointing at a different port) are honored automatically. Legacy `SIM_SERVER_URL` is entirely absent from the example; the bootstrap script still accepts it as a deprecated translate-and-warn fallback for one more wave (plan 05 removes that too).
- Log strings updated: `"Roboclaws MCP server listening on %s:%s (Gateway route http://host.docker.internal:%s/mcp)"` and `"teardown: stopping MCP server"`.
- `run_result_json["sim_server_metrics"]` JSON key preserved verbatim — comment inline documents that the name is frozen for report.html compat even though the backing server is MCP. Value comes from `mcp_server.snapshot_metrics()` which plan 01 pinned as the same 8-key contract.
- Test mocks swapped from `SimHTTPServer` class mock to `make_roboclaws_mcp` factory mock. New `_make_fake_mcp_server(snapshot_metrics=None)` helper returns a `MagicMock(spec_set=[...])` pre-configured with sensible defaults matching plan 01's 8-key snapshot_metrics contract — keeps each test case's setup one line.
- Happy-path test additionally asserts `env["ROBOCLAWS_MCP_URL"] == "http://host.docker.internal:18788/mcp"` and `"SIM_SERVER_URL" not in env` — pins the "example drops the legacy var entirely" contract.
- New `test_roboclaws_mcp_url_env_override_is_honored` — pre-sets the env var via `patch.dict`, runs the example, asserts the bootstrap subprocess receives the operator value verbatim. Guards threat T-02.6-23 (regression from `setdefault` back to `=`).

## Decisions Made

1. **Kickoff prompt delegates to SKILL.md, doesn't duplicate it.** Plan 03 already shrunk SKILL.md to the essential loop + interjection protocol; restating any of that in the kickoff prompt would create a drift surface. The kickoff says "Read SKILL.md and follow it exactly" and adds only the run-scoped context (budget, wall-clock, human_message acknowledgment). Five non-empty lines cover it.
2. **No "if X fails, try Y" fallback language in the prompt.** Per the plan's explicit `<action>` guidance: under `profile: minimal` the agent has exactly `session_status` + three `roboclaws__` tools. There are no fallbacks to suggest. Language like "if observe is unavailable, try curl" is exactly what let Kimi drift back to `exec` in the Phase 2.5 runs (see SPIKE-FINDINGS.md U2).
3. **Single `env.setdefault` line, no ROBOCLAWS_TOOL_PROFILE mutation.** The example has no reason to care whether the operator picked `minimal` or `coding`; plan 02 set the default and validates input. Example stays minimal (no pun) and passes whatever environment it receives to the bootstrap subprocess.
4. **`run_result_json["sim_server_metrics"]` JSON key preserved verbatim.** The name is frozen by plan 01's decision log and the trace_schema_reference.json fixture. Renaming would force churn in report.html + render_autonomous_replay.py + any operator diffs. The 8-key snapshot_metrics contract is stable across the HTTP -> MCP swap; the JSON shape doesn't change. Inline comment at the emission site explains the name mismatch so the next reader doesn't chase it.
5. **Test fake uses `spec_set=[...]` rather than bare `MagicMock()`.** The example touches exactly 5 methods + 2 attributes on the MCP server; listing them explicitly means a future edit that calls `mcp_server.start()` (doesn't exist) or `mcp_server.ready.wait()` (doesn't exist) fails with `AttributeError` at the test boundary instead of silently no-opping on a MagicMock and passing false green.
6. **Simplified the override test's SIM_SERVER_URL assertion.** The plan asked for "pre-set env var and assert the example does NOT overwrite it" — the straightforward assertion is `captured_env.get("ROBOCLAWS_MCP_URL") == override_url`. A second assertion checks that the legacy loopback URL `http://host.docker.internal:18788` isn't present in `SIM_SERVER_URL`; an inherited pre-existing value (e.g. if CI sets `SIM_SERVER_URL` for some other reason) is tolerated because the example itself no longer sets it.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Initial override-test assertion was non-sensical chained comparison**

- **Found during:** Task 3 writing the `test_roboclaws_mcp_url_env_override_is_honored` test
- **Issue:** First draft wrote `assert "SIM_SERVER_URL" not in captured_env or captured_env.get("SIM_SERVER_URL") == captured_env.get("SIM_SERVER_URL") != "http://host.docker.internal:18788"`. The chained `==` then `!=` comparison is logically valid Python but non-sensical (comparing a value to itself). The intent was "it's fine if the key is absent OR if it's present but not the legacy loopback URL".
- **Fix:** Simplified to `assert captured_env.get("SIM_SERVER_URL") != "http://host.docker.internal:18788"`. `.get()` returns `None` if absent, which is `!= "http://..."` by construction, so the single-line form covers both cases.
- **Files modified:** `tests/test_openclaw_nav_autonomous.py` (1 assertion)
- **Verification:** Test still passes after simplification; intent is clearer.
- **Committed in:** `c662733` (the bad chained version never left my working tree — caught on review before staging).
