## Decisions Made

1. **Retrospective focuses on surprises only, not recap.** Every shipped fact already has a SUMMARY.md under `.planning/milestones/v1.98-phases/02.6-.../`. Duplicating them in the retro would age the same way the spike's 0.408 ratio aged. The retrospective's value is the narrative that *would* be lost to git archaeology — the "why we changed our minds about X" threads. Followed `phase-2.3.md` (one-page decline narrative) and the `phase-2.2.md` retrospective section (lessons-only) as templates.

2. **Kept the deprecated-SIM_SERVER_URL line in local.md** even though the plan's behavior block says "Any pre-existing prose referencing `SIM_SERVER_URL` (as the primary)…". One mention remains, explicitly labeled as a deprecated fallback, because bootstrap's translate-and-warn path still exists until the next phase removes it. Operators with stale shells benefit from the grace-note pointer; scrubbing it would be dishonest about current behavior. The plan's constraint was "as the primary" — it is no longer the primary.

3. **Restored the Linux `--network host` fallback paragraph** in the new MCP section. That fallback is still operationally valid — the new MCP contract changed the tool surface, not the networking reality. Moved it to the end of the new section where the original Phase 2.5 section had it, compressed to two sentences.

4. **Did not rewrite Phase 2.2 push-model sections.** Per plan 01 CONTEXT.md domain boundary ("Replacing push-model code paths used by Phase 2.2 territory/coverage games. This phase is additive."). Territory / coverage / per-agent-SOULs sections in local.md are untouched.

5. **Used verbatim source-pointer lines in internals doc.** `/app/dist/pi-bundle-mcp-tools-CxZ16DeR.js:128-170` (spike F-1) and `/app/dist/tool-policy-CwZyJfFc.js:352-354` (U2 confirmation) are reproducible pointers against the pinned image; future image bumps will want exactly these anchors to re-derive the loader shape + profile values. Less durable summary prose would hurt future spelunkers.

6. **Orchestrator prompt adjustment.** The plan file defines 2 tasks (local.md + internals.md), but the orchestrator prompt added a third must-have (the retrospective). Handled by treating it as an in-scope deliverable and committing it under the same `docs(phase-02.6-07):` prefix as the other two. Its pattern matches existing retros; no plan structure violated.
