## Decisions Made

1. **JPEG-b64 inside trace.jsonl, PNG for the MCP tool result.** The spike (F-4) proved `Image(data=<png-bytes>, format="png")` is the shape the Gateway forwards to Kimi multimodal. But `scripts/render_autonomous_replay.py` reads `fpv` / `overhead` out of `trace.jsonl` as JPEG base64 today. Keeping the trace fields JPEG-b64 avoids rewriting the renderer in this plan (deferred to whichever plan modernizes reports). The MCP tool result uses PNG.
2. **Tool closures delegate to private `_do_*` methods.** Tests drive the logic directly without binding a port. Also keeps the HTTP layer mocking-free for future CI — we test behavior, not transport. Pattern documented so plan 03's example follows the same shape.
3. **`snapshot_metrics` equality, not superset.** Downstream tests (`tests/test_openclaw_nav_autonomous.py`) reference `run_result_json["sim_server_metrics"]["observed_once"]` and siblings by exact name. Superset would silently let a key rename through. Task 4 test enforces set equality.
4. **`close()` is defensive, not reliable.** FastMCP in `mcp==1.27.x` has no documented shutdown hook; the daemon-thread flag is the safety net. Good enough for single-server local-dev (the rest of the phase targets exactly that).
5. **Add `openclaw` extra, not a top-level dependency.** Mirrors `ai2thor` living in `dev` not base — the core library stays installable without the Gateway path (important for CI's `lint-and-mock` job).
