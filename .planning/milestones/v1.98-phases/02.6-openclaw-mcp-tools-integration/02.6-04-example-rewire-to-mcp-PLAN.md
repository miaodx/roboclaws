<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Example script ↔ MCP server (in-process) | Same Python process — no network trust boundary between them. |
| Example script ↔ Gateway container | Kickoff `/v1/chat/completions` call over HTTP to localhost:18789 with a bearer token; existing bridge handles this. |
| Example script ↔ stdin | The example reads interjection lines from stdin and feeds them into the MCP server's human_message queue. Stdin is the developer's terminal — same trust as the shell that launched the script. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.6-13 | Spoofing | Kickoff prompt drifts back to mentioning exec/curl in a future edit | mitigate | Task 1 verify step greps for 8 forbidden substrings in the prompt. A regression would fail CI. |
| T-02.6-14 | Tampering | Stdin line is treated as trusted input | accept | Stdin comes from the developer's terminal; free-form human_message text ends up in trace.jsonl and in the agent's next observe result. Agent is instructed (via SKILL.md) to acknowledge + act on it; no code execution path. Risk accepted. |
| T-02.6-15 | Denial of Service | mcp_server.run_in_thread() fails to bind port 18788 (already in use) | mitigate | FastMCP raises OSError on bind failure; caller's try/finally tears down engine cleanly.
