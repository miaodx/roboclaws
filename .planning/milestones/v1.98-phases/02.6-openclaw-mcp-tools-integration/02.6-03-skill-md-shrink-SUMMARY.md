## Decisions Made

- **Body-only budget measured via awk extractor, not `wc -l`.** The D-06 rule locks the body at ≤25 lines where "body" = content after the second `---` fence. Using `wc -l` on the whole file would conflate frontmatter + body and mis-measure budget fits. The plan's verification command uses the same extractor — matching it here makes CI-vs-local results identical.
- **Dropped the optional SOULs pointer.** The plan allowed a one-line pointer to `souls/aggressive.md` etc. "only if the body-line extractor still reports ≤25 with it included". Skipped because: (1) SOULs are loaded by the bootstrap into `SOUL.md`, not discovered by the agent through this skill file, so the pointer is decorative; (2) keeping the body at 10 non-blank lines leaves plenty of headroom for a future human_message or loop-pattern refinement without pushing over budget.
- **Kept the frontmatter fence count at exactly 2.** The Gateway skill loader relies on the `---` / `---` frontmatter shape. `grep -c "^---$"` returns 2; any change to that count would silently break skill loading at session start.
- **Description change limited to the final clause.** Changed "…return a single navigation action as JSON" → "…return a single navigation action via MCP tools". Kept the "Navigate a simulated robot…AI2-THOR indoor scene" lead phrase so the skill-routing description still fires on the right asks (AI2-THOR + nav) and skips the wrong ones (generic chat, coding).
