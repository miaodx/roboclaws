# Phase 02.6 Plan 02: Bootstrap Seeds MCP and Profile Summary

**Bootstrap writes `mcp.servers.roboclaws = {transport: streamable-http, url: ROBOCLAWS_MCP_URL}` + `agents.list[*].tools.profile = minimal` into openclaw.json before `docker run` — avoiding the SIGUSR1 restart dance that spike F-3 proved.**

## Accomplishments

- `scripts/openclaw-bootstrap.sh` now seeds the MCP server block and the per-agent `tools.profile` block into `openclaw.json` BEFORE the Gateway container starts, so the Gateway never triggers the SIGUSR1 → PID-1-exit cycle documented in spike F-3.
- New `ROBOCLAWS_MCP_URL` env var (default `http://host.docker.internal:18788/mcp`) is the authoritative source for the seeded MCP URL; legacy `SIM_SERVER_URL` still works as a translate-and-warn fallback for one wave.
- New `ROBOCLAWS_TOOL_PROFILE` env var (default `minimal`) gates the agent tool surface; typos die 1 at bootstrap start, preventing silent broadening of the tool-surface attack surface (threat T-02.6-06).
- `_run_preseed(tmp_path, env_overrides)` helper in `tests/test_openclaw_bootstrap.py` runs the pre-seed python heredoc against a tmp config root without Docker. Five new regression tests pin the contract:
  - Transport key is `transport` not `type` + value is `streamable-http` (spike F-1 guard).
  - Every agent carries `tools.profile = minimal` with no sibling keys (D-03 guard).
  - `ROBOCLAWS_MCP_URL` env override is honored.
  - `ROBOCLAWS_TOOL_PROFILE=coding` env override is honored across all agents.
  - Re-running the heredoc produces byte-identical `openclaw.json` (idempotency guard).

## Decisions Made

- **Default ROBOCLAWS_MCP_URL uses `host.docker.internal:18788/mcp`**, matching the existing `--add-host=host.docker.internal:host-gateway` network plumbing. Container → host loopback is already in place; no new Docker networking needed.
- **SIM_SERVER_URL deprecation is translate-and-warn, not hard-fail.** A one-wave grace period (until plan 05) lets local operators and CI scripts keep working even if they hadn't seen the rename; the stderr `WARN: SIM_SERVER_URL is deprecated` line is the visible cue.
- **Validation list is the three Gateway-accepted profile values** (`minimal | coding | messaging`, per spike F-2). Accepting arbitrary values would let a typo silently broaden the surface.
- **Line-based heredoc extraction in tests**, not regex. The outer `sh -lc '...'` quoting of the heredoc (`<<'"'"'PY'"'"'`) is nasty to regex; scanning for a line containing `python3 - <<` + `'PY'` and the matching `PY`-only line is unambiguous and fails loudly when the script structure changes.
- **`main` fallback agent is left without tools.profile** — it exists because the Gateway insists on a default agent existing, but no request ever routes to it. Injecting a profile there would add untested state.
